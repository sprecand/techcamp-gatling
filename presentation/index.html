JMeter

Source:Dall-E (Prompt: «Old lady sitting in an armchair, having a cup of tea and putting stamps on a huge pile of letters»)

https://jmeter.apache.org/
  • Grand Old Lady of Load Testing: Apache Open-Source Projekt, lange gewachsen mit grosser Community, viele Plugins etc
  • Standalone Applikation, kein Maven Plugin o.Ä., keine CI/CD Integration
  • nicht nur für Java verwendbar
  • Load testing mit UI (ist in Swift geschrieben => Look&Feel nicht State of the Art)
  • ‘Headless Mode’ verfügbar (CLI)
  • Output: XML (versionierbar)
  • Beschränkte Möglichkeit für Scripting
  • Unterstützt viele Protokolle, wie FTP, SOAP, HTTP, etc

Gatling FTW?

https://gatling.io/

  • Load Testing as Code
  • integrations
  • reporting as HTML

Gatling Enterprise

  • Cloud offering (Load Testing as a Service)
  • Requests von verschiedenen Locations möglich
  • Trends möglich --> Regressionstests

K6? Chame das ässe?

https://grafana.com/docs/k6

  • Tests as Code (JS, experimental TS) (Parallel users implemented in Golang (Goroutines))
  • Much more performant than frameworks running on the jvm
  • Detaillierte Scenarios (ramp-up, etc…) möglich
  • Verschiedene Outputformate (JSON, CSV, Kafka, TimeSeries in InfluxDB, etc…)
  • Test gegen SLO möglich (z.B. “99% der Requests dauern weniger als 10ms”)
  • Viele built-in metrics, Custom Metrics möglich
  • Regressionstests mit Trends möglich
  • Gut dokumentiert mit vielen Examples
  • *aaS möglich (Grafana Cloud K6)
  • Lokal ist ein Umweg über eine unterstützte Datasource (zB InfluxDB) nötig
  • Debugging nicht möglich, unterstützt nur modernere Standards (kein Support für FTP o.Ä.)

Grafana Cloud K6

  • Entweder local execution streamed in die cloud oder direkt cloud execution
  • Cloud execution scheduled, fully customizable mit der AWS region von wo man die execution starten will
  • Gibt insights über failure rates und best practices beim Aufbau der test cases (KI-Unterstützung)
  • K6 Test Builder ist nur verfügbar über Grafana Cloud (zusammenklicken von test cases statt js code)
